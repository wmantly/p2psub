p2psub

Mesh Peer-to-Peer JSON Pub/Sub with Zero Dependencies

Features

Mesh Network

Messages forward to all connected peers, even if not directly connected

Dynamic Peers

Add and remove peers on the fly with automatic reconnection

RegExp Patterns

Subscribe to topics using powerful regular expression patterns

Modular Design

Separate P2P and PubSub classes for maximum customization

Zero Dependencies

No external runtime dependencies, pure Node.js

Easy to Use

Simple API with CLI support for quick relay setup

Quick Start

Standalone PubSub (No Networking)

For simple in-process pub/sub without P2P networking, use the singleton ps instance:

const {ps} = require('p2psub');

// Subscribe to topics
ps.subscribe('my-topic', (data) => {
    console.log('Received:', data);
});

// Publish to topics
ps.publish('my-topic', {message: 'Hello World'});

Or create your own isolated PubSub instance:

const {PubSub} = require('p2psub');

const myPubSub = new PubSub();
myPubSub.subscribe('topic', callback);
myPubSub.publish('topic', data);

P2PSub (With Mesh Networking)

Create a P2PSub instance and start publishing and subscribing across the network:

const {P2PSub} = require('p2psub');

const p2p = new P2PSub({
    listenPort: 7575,
    peers: [
        '10.10.10.11:7575',
        '10.10.10.12:7575'
    ]
});

// Subscribe to a topic
p2p.subscribe('announcement', (data) => {
    console.log(data.message);
});

// Publish to the network
p2p.publish('announcement', {
    message: 'Hello, mesh network!'
});

Pattern Matching

Use RegExp patterns to match multiple topics:

// Match all topics starting with 'announcement'
p2p.subscribe(/^announcement/, (data, topic) => {
    console.log(topic, data.message);
});

// Match all topics
p2p.subscribe(/./, (data, topic) => {
    console.log(topic, data);
});

Configuration Options

listenPort (Number or String, optional)
  • Sets the incoming TCP port for the local peer
  • If omitted, peer will not accept incoming connections
peers (Array of Strings, optional)
  • List of peers to connect with in format {host}:{port}
  • Example: ['192.168.1.1:7575', 'example.com:8080']
logLevel (Array of Strings or false, optional)
  • Controls logging: ['info', 'warn', 'error']
  • Set to false to suppress all messages
preBroadcast(data, topic) (Function, optional)
  • Filter or modify data before network broadcast
  • Return modified object or false to prevent propagation

API Reference

Module Exports

P2PSub

Combined P2P + PubSub class for mesh network pub/sub

P2P

Standalone peer-to-peer networking class

PubSub

Standalone pub/sub class (no networking)

ps

Singleton PubSub instance for convenience - ready to use without instantiation

P2PSub Methods

subscribe(topic, callback)

Subscribe to a topic using a String or RegExp pattern

publish(topic, data)

Publish data to a topic across the network

addPeer(peer)

Dynamically add a peer connection

removePeer(peer)

Disconnect and remove a peer
